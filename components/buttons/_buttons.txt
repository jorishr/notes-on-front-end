#####################
I.	A-TAG AS BUTTON
#####################

This is the preferred way of creating buttons.

Always set display: inline-block, especially if there is vertical padding. By setting the inline-block, other elements around will be aware of the vertical padding.

HTML: add class .btn or .button
If not part of a form: add attribute role="button"

REMOVE ALL STYLING
	.btn { all: unset } 
This will select all possible properties and set back to default values: initial or inherit. For example the font-size may be inherited from the container div but the bg color of the padding wil be set to initial value: none; so the btn is reduced to plain text.

III.	STYLING AND ANIMATING BUTTONS
###################################

A.	BASIC STYLING 

NOTE: the anchor tag is an INLINE element. Use display: inline-block to make it behave correctly if what you want is a stand-alone button.

text-transform: uppercase;
letterspacing: .08em; | .1em;

Text inside buttons may start to wrap on a second line when 	space is restricted. This may result in an ugly button. To 	avoid this you can use:

 	in css: 		.class {white-space: nowrap;}
	in bootstrap: 	text-nowrap class;

padding: .5em 1.25em;
box-shadow: 0 12px 12px -6px rgba(black, .3)

	None on the X-axis; 12px on the Y-axis; blur: none; 	spread: -6px;

A negative value for the spread reduce the length of the shadow. A good rule of thumb is to use half of the offset.

When working with TWO buttons next to each other make one have a BACKGROUND and the other a BORDER but no background, to have a contrasting OUTLINE effect.

B.	USER EXPERIENCE/ANIMATION

Set opacity at .75, when :hover opacity: 1.
transform: scale(1); when :hover transform: scale(1.05);
tranistion: 
	transform ease-in-out 250ms, 
	box-shadow ease-in-out 250ms;
