1.	ID AND CLASS
##################

	ID 

Een id of "hook" kan slechts 1 keer gebruikt worden per pagina. Het is een HTML attribuut id="" dat in CSS wordt geselecteerd met  #id {...}

	CLASS

Een class wordt gebruikt voor meerdere elementen met een of meerdere gedeelde stijlkenmerken. In HTML attribuut class="", in CSS selecteer je .<class>

Het wordt veelal aangeraden om ID's te vermijden en alles een class te geven maar wanneer frameworks zoals Bootstrap of backend JS frameworks als Express worden gebruik ga je toch ID's gebruiken om elementen te identificeren en te selecteren.

Je kan dan ook id en class attributen combineren en de kenmerken die gemeenschappelijk zijn via class definieren en aparte stijlen aangeven via een ID. 

BEM conventie daarentegen steunt op classes, child classes and modifier classes. 


########################
II.	ADVANCED SELECTORS
########################

h1,h2,strong {font-weigth: 900}	// group common properties at the 						beginning of each style sheet

div.class	// selecteer alleen de element met .class die ook een 			div zijn. Dus a.class wordt niet geselecteerd.

h3 ~ p 	// selecteer alle p's in een div waarin ook een h3 zit

Adjescent selector
h3 + p 	// selecteer alleen de p die net na een h3 komen

div > p 	// selecteer alle p where the parent is a <div> 

	PSEUDO CLASS SELECTORS

p:first-letter {...}	// select the first letter of every <p>
p:first-line {...}	// select the first line of every <p>

col:first-child{background-color:red;}

h3:hover {...} 		// on hover

input:checked {...} 	// when checked

.btn:active {...} 	// when active

section:not(.first):nth-child(odd){...)
				
				// alle oneven secties krijgen een bepaalde 				achtergrondkleur, behalve de eerste.

	ALTERNATION WITH PSEUDO CLASSES

section:nth-child(odd) {...}
section:nth-child(even) {...}

col:nth-child(2n+3){...}

	(2n+3) 	// elke tweede kolom, te beginnen vanaf de derde.
	(4n+1)	// elke vierde, te beginnen vanaf de eerste

	ATTRIBUTE SELECTORS

a[attribuut] 	// selecteer <a>'s met een bepaald atrribuut

	input[name="email"]{...}

	a[href="https://wwww.google.com"] 
	a[target] 
	a[tartget="_blank"]

You can add regular expression to this syntax:

a[href*="google.com"] // all links that contain the string

p[name^="text"]{...} 	// all strings that start with "text"
p[name$="text"]{...} 	// all strings that end with "text"


#################
III.	SPECIFICITY
#################

span {...} // includes all span elements, thus low specificity

div span {...} 	// overrides simple span

div#test span {...} 	// most specific

A directly targeted element has priority over INHERETED STYLES. In the example below the h1 inherits color from body but the directly target h1 overrides the inherited properties.

	body id="parent" 
	
	div#parent {color:red;}

	h1 {color:green;}

Daarom zet bovenaan in je CSS document de algemene stijlen voor h1, p, a, etc. Die worden dan toegepast in alle secties van je pagina, tenzij je met een meer specifieke selector, een andere stijl aanbrengt.
